# owner 业主系统

这是在同一个系统中，支持多个业主同时提供服务；

## 场景和问题

### 将只属于某个业主的数据独立保存到其他数据源

想不到

### 用户，商品，微信公众号信息，短信通道只属于业主

数据层面，数据表需要新增字段维护它的业主关系。

服务层面，应该存在高可用（静态获取）的方式获取当前业主。

## 传说中的店中店？

具备创建子业主的功能？

## 部署机制上更加可控

顶级业主（包括单业主的情况），允许将信息委托在(业主托管平台)上

即便是单业主系统，也应该建立本地相对应数据表。

## 如何识别业主

1. 允许API设定某个会话锁定到某业主
1. 允许通过域名
